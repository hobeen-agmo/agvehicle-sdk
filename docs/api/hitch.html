AgVehicle SDK / 도메인 모델 / :hitch

히치 (:hitch)

히치 위치를 읽고(구독) 제어(0~100%)한다. 표준 계열이며 J1939/ISOBUS 재작성이 가능하다. 읽기는 자격이 필요 없고, 쓰기는 매니페스트에 USES_CONTROL 선언이 필요하다. raw↔% 변환은 SDK가 숨긴다.

읽기

위치 구독

Signal을 상속해 lifecycle.addObserver(...) 한 줄로 ON_START에 자동 구독, ON_STOP에 자동 해제된다.

Hitch(context: Context, intervalMs: Long = 0) read farm.agmo.vehicle.hitch

등록 방식은 두 가지이며, 두 방식은 기능적으로 동등하다.

방식생성 · 등록
람다Hitch(context, intervalMs) + hitch.onPosition { } — 필요한 신호만, 체이닝 가능
리스너Hitch(context, listener: Listener, intervalMs) — 여러 신호를 한 객체에서 override

각 콜백마다 대응하는 람다 메서드가 있다. onPosition { }, onRearHitch { }, onFrontHitch { } 등이 있으며, 아래 Listener 표의 on… 이름과 1:1로 대응한다. 람다 방식은 등록한 신호만 구독한다.

intervalMs는 앱 콜백의 최소 간격(ms)을 지정하며 기본값은 0이다. 0이면 값이 올 때마다 전달하고, 0보다 크면 그 간격마다 최신값을 한 번만 전달한다(core.html#throttle 참고).

Hitch.Listener

콜백인자설명
onPositionHitchPosition히치 위치(%) 갱신 — 필요한 콜백만 override, 나머지는 no-op
onRearHitchRearHitch후방 히치 상세(작업중·링크힘·드래프트) 갱신
onFrontHitchFrontHitch전방 히치 위치/작업 상태 갱신
 
val hitch = Hitch(this, intervalMs = 200)
hitch.onPosition { p -> runOnUiThread { render(p.percent) } }
lifecycle.addObserver(hitch)     // ON_START 자동 구독 / ON_STOP 자동 해제

여러 신호를 한 객체에서 다루려면 아래와 같이 리스너 방식으로 등록할 수도 있다. 두 방식은 기능적으로 동등하다.

val hitch = Hitch(this, object : Hitch.Listener {
  override fun onPosition(p: HitchPosition) = runOnUiThread { render(p.percent) }
}, intervalMs = 200)
lifecycle.addObserver(hitch)     // ON_START 자동 구독 / ON_STOP 자동 해제

쓰기

제어 — 히치 올리기/내리기

읽기(구독)와 달리 제어는 단일 세션(토큰, 선점)이라 생명주기가 아닌 세션 핸들로 다룬다. Hitch.control()이 null이면 자격 없음(매니페스트 미선언), 상위 계층 보유 중, 미연결 중 하나다.

필수 제어를 쓰려면 앱 매니페스트에 USES_CONTROL = HITCH_CMD 선언이 있어야 한다. 자세한 자격, 실행레벨 규칙은 정책 문서를 참고한다.
Hitch.control(context: Context): HitchControl? ctrl PGN 0xEF80 · key HITCH_CMD farm.agmo.vehicle.hitch

히치 제어권을 잡는다. 반환이 null이면 자격 없음, 상위 보유 중, 미연결 중 하나다. 즉 세션을 얻지 못했다는 뜻이다.

필드타입단위설명
CONTROL_KEYString–"HITCH_CMD" — 데몬 내장 제어 신호 key

메서드타입단위설명

val ctrl = Hitch.control(this) ?: return     // 매니페스트 USES_CONTROL 필요
ctrl.onLost { runOnUiThread { lockUi() } }         // 상위 계층이 뺏으면
ctrl.setPosition(100.0)                            // % — raw 변환은 SDK가 처리
ctrl.release()                                     // 정상 반납
