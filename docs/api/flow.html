Flow (:flow)

위 도메인 신호들을 Kotlin Flow로 받는 옵션 모듈이다. 콜백 코어(Signal/Listener)는 코루틴 없이 돌지만, 코루틴을 쓰는 앱은 이 모듈 하나만 추가하면 스트림별 구독과 조합을 얻는다. SDK에서 코루틴 의존은 여기뿐이다(Room의 room-ktx와 같은 계층 분리).

개념

동작 방식

각 Flow는 core의 subscribeMessage/subscribe를 callbackFlow로 감싼 것이다. 콜드 스트림이라 마지막 수집자가 사라지면 awaitClose에서 구독을 해제한다. buffer(1, BufferOverflow.DROP_OLDEST)를 걸어 고빈도 신호(IMU 100Hz 등)에 수집이 밀려도 최신값만 유지한다(프레임 밀림 방지).

범위 :flow는 기존 도메인 모듈(:imu · :engine · :hitch · :vehicle)의 데이터 클래스를 그대로 흘려보낸다. 새 필드는 없다. 필드와 단위는 각 도메인 문서를 참고한다.

사용 예

repeatOnLifecycle · combine

표준 안드로이드 생명주기 스코프 안에서 필요한 스트림만 collect한다. 조합도 공짜다. 서로 다른 도메인의 Flow를 combine으로 묶어 대시보드 하나로 만들 수 있다.

lifecycleScope.launch {
  repeatOnLifecycle(Lifecycle.State.STARTED) {           // 표준 생명주기
    ImuFlow.angles(context).collect { render(it) }       // 필요한 스트림만
  }
}
// 조합도 공짜
combine(ImuFlow.angles(ctx), EngineFlow.eec1(ctx)) { a, e -> Dashboard(a, e) }.collect { … }
주의 repeatOnLifecycle 밖(예: 무조건 launch)에서 collect하면 화면이 백그라운드로 가도 구독이 살아있다. 배터리와 바인더 트래픽이 낭비된다. 반드시 생명주기 스코프 안에서 collect해야 한다.
