:oem-genericio

🏭 AGMO 제조사 고유 모듈이다. 범용 CAN IO 보드(아날로그/디지털 입력, PWM/LED 출력, ID 0x200~0x204)를 추상화한다. 하드웨어 자체는 특정 제조사 CAN에 고유하지 않지만, 소스가 NEVONEX GenericIO를 참조하므로 provenance 구분을 위해 oem 계열에 둔다.

사용법

빠른 시작

필요한 신호만 onXxx { } 람다로 등록한다. object나 override 없이 등록할 수 있고, 체이닝도 가능하다. 채널 기반 신호(onAnalogIn/onDigitalIn)는 콜백 인자가 (channel, value) 2개다. intervalMs로 앱 콜백의 최소 간격(ms)을 조절한다.

val io = GenericIo(this, intervalMs = 200)
io.onAnalogIn { ch, v -> runOnUiThread { render(ch, v) } }
  .onSupply   { main, r5 -> runOnUiThread { render(main, r5) } }
lifecycle.addObserver(io)

여러 신호를 한 객체에서 다루려면 아래와 같이 리스너 방식으로 등록할 수도 있다. 두 방식은 기능적으로 동등하다.

// 입력 읽기 (자격 불필요)
val io = GenericIo(this, object : GenericIo.Listener {
    override fun onAnalogIn(ch: Int, value: Double) = runOnUiThread { render(ch, value) }
    override fun onDigitalIn(ch: Int, on: Boolean)  = runOnUiThread { render(ch, on) }
}, intervalMs = 200)
lifecycle.addObserver(io)

// PWM 출력 제어 (매니페스트에 USES_CONTROL 선언 필요)
val pwm = GenericIo.pwm(this, side = "HS", channel = 1) ?: return
pwm.setDuty(50.0)                       // 0~100%
// LED 출력 제어 (RGBW 한 프레임에 결합 송신)
val led = GenericIo.led(this, led = 1) ?: return
led.setColor(channel = 0, r = 255, g = 0, b = 0, w = 0)
GenericIoSignals

신호 키 — 전체

GenericIoSignals object는 모든 키(또는 키 생성 함수)를 노출한다. channel은 1-기반이다.

키 / 함수방향ID설명
analogIn(channel: Int)읽기0x200genericio:AnalogIN<channel> — 아날로그 입력, channel 1~4
digitalIn(channel: Int)읽기0x201genericio:DigitalIn<channel> — 디지털 입력, channel 1~4
SUPPLY_VOLTAGE읽기0x201genericio:AnalogSupplyVoltage — 공급전압(메인)
SUPPLY_5V읽기0x201genericio:AnalogSupply5V — 5V 레일 전압
pwm(side: String, channel: Int)쓰기0x202genericio:PWM_<side><channel> — PWM 출력, side="HS"/"LS", channel 1~2
led(led: Int)쓰기0x203 / 0x204genericio:LED<led> — LED RGBW 결합 명령, led 1(0x203)/2(0x204)
API

읽기 — GenericIo.Listener

GenericIo(context: Context, intervalMs: Long = 0) read :oem-genericio

등록 방식은 두 가지이며, 둘은 동등하다.

방식생성 · 등록
람다GenericIo(context, intervalMs) + io.onAnalogIn { } — 필요한 신호만, 체이닝 가능
리스너GenericIo(context, listener: Listener, intervalMs) — 여러 신호를 한 객체에서 override

각 콜백마다 대응하는 람다 메서드가 있다. onAnalogIn { }, onDigitalIn { }, onSupply { } 등이 있다. 채널 기반 신호(onAnalogIn/onDigitalIn)는 콜백 인자가 (channel, value) 2개다.

intervalMs는 앱 콜백의 최소 간격(ms)을 지정한다. 0이면 신호가 올 때마다 전부 전달하고, 0보다 크면 최대 그 간격마다 최신값을 1회만 전달한다.

필요한 콜백만 override한다. ANALOG_IN_COUNT/DIGITAL_IN_COUNT는 각 4이다. 공급전압 두 값은 한 프레임(0x201)이라 onSupply 하나로 함께 온다.

interface Listener {
    fun onAnalogIn(channel: Int, value: Double) {}
    fun onDigitalIn(channel: Int, on: Boolean) {}
    fun onSupply(mainVolts: Double, rail5V: Double) {}
}
제어

쓰기 — PWM / LED 출력

제어 세션은 GenericIo.pwm(context, side, channel) 또는 GenericIo.led(context, led) 컴패니언 함수로 얻는다. null이 반환되면 자격이 없거나, 세션을 이미 보유 중이거나, 미연결이거나, 파라미터가 잘못됐다(side, channel 범위 밖)는 뜻이다. 매니페스트에 USES_CONTROL 선언이 필요하다.

PwmOutput ctrl genericio 0x202

PWM 듀티(0~100%)를 제어한다.

메서드반환설명
setDuty(percent: Double)Boolean듀티 지령(0~100, 범위 밖은 clamp). false=세션 상실
onLost(cb: () -> Unit)Unit제어권 상실 콜백
release()Unit제어권 정상 반납
stopAndRelease()Unit정지 지령 후 반납
한계 PWM 4채널이 한 프레임(0x202)이라, 채널 하나만 지령해도 나머지 채널 바이트는 0으로 함께 나간다. 다채널 동시 제어가 필요하면 다중필드 write 또는 결합 신호로의 확장이 필요하다.
LedOutput ctrl genericio 0x203 / 0x204

LED RGBW를 제어한다. Channel과 RGBW를 결합 신호로 묶어 한 프레임에 원자적으로 송신한다. LED1(0x203)은 Channel(bit0)+R(32)+G(40)+B(48)+W(56)의 64bit 결합값, LED2(0x204)는 Channel(bit0)+R(8)+G(16)+B(24)+W(32)의 40bit 결합값으로 인코딩된다.

메서드반환설명
setColor(channel: Int, r: Int, g: Int, b: Int, w: Int)BooleanRGBW+채널 색 지령. false=세션 상실
onLost(cb: () -> Unit)Unit제어권 상실 콜백
release()Unit제어권 정상 반납
stopAndRelease()Unit정지 지령 후 반납
한계 LED1은 64bit 결합값인데 현재 명령 경로가 double(52bit 정밀도)을 거치므로, 상위 비트(Blue<<48/White<<56)에서 정밀도 손실이 발생할 수 있다.
