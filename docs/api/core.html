핵심 (core)

AgVehicle 하나가 서비스와의 프로세스 단일 연결을 쥐고, key 기반 구독과 제어를 도메인 모듈(engine/hitch/…)과 앱에 멀티플렉싱해준다. 타입 있는 신호 클래스가 필요 없는 동적 신호나, 아직 도메인 모듈이 없는 신호는 여기 API로 직접 다룬다.

연결

AgVehicle · 연결

서비스는 uid당 콜백 하나만 기억하므로(clients[uid]), 도메인 모듈이 각자 bindService하면 서로의 콜백을 덮어쓴다. 그래서 프로세스당 shared 연결 하나만 두고, 그 위에 key별 콜백을 등록해 나눠 쓴다.

AgVehicle.shared(context)read:core

프로세스 공유 연결이다. 첫 호출에 bindService를 걸고, 이후 같은 인스턴스를 반환한다. 도메인 모듈과 앱이 모두 이걸 통해 서비스에 얹힌다(연결은 하나).

companion object {
  fun shared(context: Context): AgVehicle
}
파라미터타입설명

interface ConnectionListener {

onDisconnected()서비스 사망 등. SDK가 BIND_AUTO_CREATE로 자동 재바인딩을 시도한다.
onRejected()attach 거부 — 실행 충돌 차단. 앱 종료를 고려해야 한다.
onBindFailed()bindService 자체 실패(서비스 미설치 등). 재시도 여지가 없어 앱 종료를 고려해야 한다.

읽기

신호 읽기 · subscribe

같은 key를 여러 곳이 구독해도 서비스엔 SUB 1회만 나간다(refcount). 첫 구독자가 붙을 때 SUB를 보내고, 마지막 구독자가 닫힐 때 UNSUB을 한 번 보낸다. 미연결 상태에서 호출해도 되며, 연결/재연결 시 자동 반영된다.

subscribe(key, intervalMs = 0, onValue)read:core

신호 하나를 구독한다. 반환 핸들을 close()하면 해제된다.

  key: String,
  intervalMs: Long = 0,
  onValue: (SignalValue) -> Unit,
): Subscription

keyString—구독할 신호 key
intervalMsLong0앱 콜백 최소 간격(ms). 스로틀 참고
onValue(SignalValue) -> Unit—값이 갱신될 때마다(또는 게이트 통과 시) 호출
subscribeMessage(keys, intervalMs = 0, onValues)read:core

한 메시지(신호 여럿)를 함께 구독한다. 신호가 하나라도 갱신되면 그 메시지의 최신값 맵을 넘기며, 이는 도메인 모듈이 타입 있는 "ID별 클래스"로 조립하는 토대가 된다. 한 CAN 메시지의 신호들은 한 프레임으로 같이 오지만 콜백 스레드가 다를 수 있어 최신값 누적은 동기화된다. 반환 핸들은 close()로 전체 해제한다.

fun subscribeMessage(

  intervalMs: Long = 0,

파라미터타입기본값설명
keysList<String>—같이 구독할 신호 key 목록(한 메시지의 신호들)
intervalMsLong0앱 콜백 최소 간격(ms). 개별 신호는 전속력으로 받아 최신값에 누적하되, 콜백은 게이트를 통과할 때만 스냅샷 1회 호출
onValues(Map<String, Double>) -> Unit—갱신된 최신값 맵(number 기준)
onStale(key, cb)read:core
fun onStale(key: String, cb: (String) -> Unit): Subscription

신호 staleness(값 끊김) 알림을 구독한다. 값 구독(subscribe)과는 별개로 등록한다.

스로틀

갱신 스로틀 · intervalMs

subscribe/subscribeMessage의 intervalMs는 leading-edge 게이트다. 값이 들어온 순간 마지막 통과로부터 intervalMs가 지났으면(now - last >= intervalMs) 그 즉시 통과시키고, 그렇지 않으면 버린다. 타이머나 코루틴 없이 단조 시계(SystemClock.elapsedRealtime)만 비교한다.

  • intervalMs = 0(기본) — 오는 값을 전부 전달(하위호환).
  • intervalMs > 0 — 최대 그 간격마다 최신값 1회만 전달. RPM처럼 연속 갱신되는 신호는 게이트가 열릴 때마다 다음 값이 통과해 실제 ~intervalMs 간격으로 흐른다.

게이트 상태는 구독 호출마다 독립이라 신호별로 다른 주기를 걸 수 있다(예: RPM은 100ms, 다른 메시지는 다른 주기).

판정은 시각을 인자로 받는 순수 함수로 분리돼 있어 테스트 가능하다:

internal fun shouldEmit(lastMs: Long, nowMs: Long, intervalMs: Long): Boolean =
  nowMs - lastMs >= intervalMs
배경 소프트웨어 렌더 CPU 사용률을 홈 화면 234%→88%로 낮춘 최적화가 이 스로틀에서 나왔다. UI 콜백 빈도를 실제 렌더 가능한 속도로 눌러주는 게 핵심이다.
쓰기

제어 세션 · acquire

쓰기(제어)는 세션을 얻어서 한다. 토큰은 세션 안에 봉인돼 있어 앱 코드에 노출되지 않는다(오용 방지).

acquire(key, onControlLost = null)ctrl:core
fun acquire(
  key: String,
  onControlLost: (() -> Unit)? = null,
): ControlSession?

null이 반환되면 미연결, 자격 없음(매니페스트 미선언), 동급 이상이 이미 보유 중인 경우 중 하나다. onControlLost는 이 key가 상위 계층에 선점당하면 호출되며, 등록하지 않으면 무시된다.

제어 명령(raw)을 보낸다. false가 반환되면 세션을 상실한 것이며(선점 또는 연결 끊김), acquire부터 다시 해야 한다.

타입

코어 타입

SignalValueread:core

구독 신호의 값 1건. text는 데몬이 보낸 그대로("52.4 %", "1450 rpm")라 표시용으로 바로 쓸 수 있고, 계산용으로는 number/unit으로 분해해 쓴다.

data class SignalValue(val key: String, val text: String, val quality: Quality) {
  val number: Double?   // text의 첫 토큰. 수치가 아니면 null
  val unit: String?     // text의 수치 뒤 토큰. 없으면 null
}

numberDouble?파생 프로퍼티. text의 수치 부분

Subscriptionread:core

구독 핸들이며, close()로 해제한다. 도메인 모듈은 이걸 들고 있다가 stop 시 닫는다.

fun interface Subscription { fun close() }
Signalread:core

도메인 모듈(Imu/Engine/Hitch 등)의 베이스 클래스. DefaultLifecycleObserver라 lifecycle.addObserver(signal) 한 줄이면 ON_START에 자동 구독, ON_STOP에 자동 해제된다. 각 도메인은 이걸 상속해 subscriptions()에서 "무슨 메시지를 구독해 리스너에 어떻게 넘길지"만 선언하면 되고, 연결(shared), 해제, 중복 방지는 베이스가 처리한다.

abstract class Signal(context: Context) : DefaultLifecycleObserver {
  protected abstract fun subscriptions(): List<AgVehicle.Subscription>
  fun start()   // 중복 호출 안전
  fun stop()
}

수동 제어가 필요하면 addObserver 대신 start()/stop()을 직접 호출해도 된다. 코루틴 의존은 없다. Flow가 필요하면 farm.agmo.vehicle:flow 모듈을 옵션으로 추가한다.
