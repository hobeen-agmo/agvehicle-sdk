vehicle — 차량 상태

vehicle은 차속, PTO, 배터리, DPF, GPS, 변속 등 엔진과 히치를 제외한 차량 전반의 상태를 읽는 표준 도메인 모듈이다. 모든 신호가 읽기 전용이며, 쓰기(제어) API는 없다.

범위 밖 4WD와 AutoLift 제어는 이 도메인에 없다. 4WD는 실차 CAN에 매핑되어 있지 않아 지원하지 않으며, AutoLift는 조향각, FNR, 히치 조합의 앱 레벨 기능(:oem-tractor의 Tractor.autoLift)이다.
클래스

Vehicle

도메인 객체 하나로 이 페이지의 모든 신호를 구독한다. Signal을 상속하며 lifecycle.addObserver(vehicle) 한 줄로 ON_START에 구독, ON_STOP에 해제된다.

Vehicle(context: Context, intervalMs: Long = 0) read :vehicle

등록 방식은 두 가지이며, 두 방식은 기능적으로 동등하다.

방식생성 · 등록
람다Vehicle(context, intervalMs) + vehicle.onSpeed { } — 필요한 신호만, 체이닝 가능
리스너Vehicle(context, listener: Listener, intervalMs) — 여러 신호를 한 객체에서 override

콜백마다 대응하는 람다 메서드가 있다. onSpeed { }, onBattery { }, onPto { } 등이 있으며, 아래 Listener 표의 on… 이름과 1:1로 대응한다. 람다 방식은 등록한 신호만 구독한다.

intervalMs는 앱 콜백의 최소 간격(ms)을 지정한다. 0이면 신호가 올 때마다 전부 전달하고, >0이면 최대 그 간격마다 최신값을 한 번만 전달한다(core.html#throttle 참고).

콜백은 binder 스레드에서 호출되므로 UI 갱신은 runOnUiThread로 감싼다.

필요한 콜백만 override하면 된다. 나머지는 Listener 인터페이스의 기본 구현(no-op)이 처리한다.

사용 예제

필요한 신호만 onXxx { } 람다로 등록한다. object 선언이나 override 없이 등록할 수 있고, 여러 메서드를 이어서 호출(체이닝)할 수 있다.

val vehicle = Vehicle(this, intervalMs = 200)
vehicle.onSpeed { s -> runOnUiThread { render(s.kmh) } }
      .onBattery { b -> runOnUiThread { render(b.volts) } }
      .onDpf { d -> runOnUiThread { if (d.warning) showWarn() } }
lifecycle.addObserver(vehicle)   // ← 이 한 줄이 구독/해제 전부

여러 신호를 한 객체에서 다루려면 아래와 같이 리스너 방식으로 등록할 수도 있다. 두 방식은 기능적으로 동등하다.

val vehicle = Vehicle(this, object : Vehicle.Listener {
    override fun onSpeed(s: VehicleSpeed) = runOnUiThread { render(s.kmh) }
    override fun onBattery(b: Battery)    = runOnUiThread { render(b.volts) }
    override fun onDpf(d: Dpf)            = runOnUiThread { if (d.warning) showWarn() }
}, intervalMs = 200)
lifecycle.addObserver(vehicle)   // ← 이 한 줄이 구독/해제 전부
데이터 클래스

신호 클래스

각 클래스는 CAN 메시지(ID) 하나에 대응한다. KEY(단일 필드) 또는 KEYS + from()(풀 메시지)로 구독한다.

VehicleSpeedreadCCVS1 · 0xFEF1:vehicle

차량 속도를 나타낸다. CCVS1 메시지의 VEHSPEED 필드만 뽑은 단일 값이다.

필드타입단위설명
kmhDoublekm/h차속

콜백 Listener.onSpeed(speed: VehicleSpeed) · KEY "VEHSPEED"

PtoSpeedread0xFE43:vehicle

후방 PTO 회전수를 나타낸다.

필드타입단위설명
rpmDoublerpmPTO 회전수

콜백 Listener.onPto(pto: PtoSpeed) · KEY "PTO"

BatteryreadVEP1 · 0xFEF7:vehicle

필드타입단위설명
sootPercentDouble%그을음 적재율

콜백 Listener.onDpf(dpf: Dpf) · KEY "DPFSOOT"
