:oem-tractor

🏭 AGMO 제조사 고유(proprietary) 신호로, 표준 규격(J1939/ISO 11783)이 아닌 oem 네임스페이스에 속한다. 전후진(FNR), 변속(RangeShift), PTO, 유압 히치, 가속 페달의 상태 읽기·진단과 제어, 그리고 앱 레벨 AutoLift(선회·후진 시 자동 히치 상승)까지 담는다.

네임스페이스는 agmo_customized_tractor:다. 값 출처는 AGMO 1차 설계(NEVONEX Machine Interface Requirement, Customer=AGMO Inc., Requestor Dongseok Choi 2025-02-06)와 AGMO_CustomizedTractor.dbc다. 데몬이 스케일 적용한 물리값을 전달한다.

개념

개요

상태 읽기(0x4xx)는 FNR / RangeShift(SFT) / PTO / Hydraulic(HYD) / Accelerator(ACC) 다섯 메시지로 나뉘고, 각 메시지는 State/Mode(auto) 데이터 클래스와 진단(SIG1_V/SIG2_V/diagCode) 데이터 클래스가 짝을 이룬다. 제어는 서브시스템별 세션(Tractor.fnrControl 등)으로 얻는다.

4WD 미지원 AGMO 트랙터 CAN 프로토콜에는 4WD에 대응하는 신호가 없다(로컬 DBC, 설계문서, NEVONEX 추출, SeamOS 카탈로그 4곳 전수검색 0건). 런처의 FOURWD 토글은 CAN에 연결돼 있지 않으며, 이 모듈은 그에 대응하는 제어 메서드를 두지 않는다.
AutoLift 선회/후진 시 히치 자동 상승은 별도 CAN 신호가 아니라, 조향각(STEERANGLE)과 FNR을 구독해 실존 유압 히치 제어(AD_HYD_CMD)로 상승시키는 앱 레벨 기능이다. 자세한 내용은 AutoLift를 참고한다.

제어 키

TractorControlKeys는 각 *Control.set(...)이 내부적으로 지령하는 카탈로그 키다(internal object다. 앱 코드가 이 문자열을 직접 다룰 일은 없다. 타입 세이프한 Tractor 파사드의 *Control API를 쓴다). 각 *_CMD는 value(byte0) + mode(byte1) 프레임이며, SDK는 value만 지령하고 mode는 데몬 기본값 0 = Manual(수동 오버라이드)로 고정된다. 이것이 앱 제어의 정상 의미다. auto 모드 활성화는 이 setter 범위 밖이다.

트리거 로직

조향각/FNR 갱신마다 평가하는 순수 함수 shouldTrigger(OR 조합, 독립 토글):

// (onTurn && |steerRad| > 0.35 rad) || (onReverse && FNR == REVERSE)
trigger = (onTurn && steerRad?.let { abs(it) > TURN_THRESHOLD_RAD } == true) ||
          (onReverse && reverse)

when {
  trigger && !raised -> // hitch.setPosition(100.0) — 1회, 성공 시 raised=true
  !trigger            -> // raised=false — 조건 해제 시 재무장
}
실행레벨 홈 런처는 이 기능의 제어권을 CONTROL_TIER=SYSTEM으로 선언한다. 실행레벨 중재 방식은 자격 · 실행레벨(Tier)를 참고한다.
1회성 상승은 조건 성립 시 1회만 지령한다. 자동 하강은 없다. 히치를 원위치로 내리는 것은 별도 사용자 조작 또는 앱 로직 몫이다.
파사드

Tractor

Tractor(context, intervalMs)는 읽기 전용 진입점이다. Signal을 상속해 lifecycle.addObserver(tractor) 한 줄로 위 9개 메시지를 전부 구독/해제한다. 제어는 서브시스템별로 독립된 세션(*Control)을 얻어서 한다.

Tractor(context: Context, intervalMs: Long = 0)read:oem-tractor

등록 방식은 두 가지이며, 둘은 동등하다.

방식생성 · 등록
람다Tractor(context, intervalMs) + tractor.onFnr { } — 필요한 신호만, 체이닝 가능
리스너Tractor(context, listener: Listener, intervalMs) — 여러 신호를 한 객체에서 override

각 콜백마다 대응하는 람다 메서드가 있다. onFnr { }, onRangeShift { }, onPto { }, onHydraulic { }, onAccelerator { }, onFnrDiag { }, onRangeShiftDiag { }, onHydraulicDiag { }, onAcceleratorDiag { }, onStale { } 등이 있으며, 아래 표의 on… 이름과 1:1로 대응한다. 람다 방식은 등록한 신호만 구독한다.

intervalMs는 앱 콜백의 최소 간격(ms)을 지정한다. 0이면 신호가 올 때마다 모두 전달하고, >0이면 최대 그 간격마다 최신값을 한 번만 전달한다.

필요한 콜백만 override하면 된다. 모든 메서드는 기본 구현(no-op)이 있다.

필드타입단위설명
onFnr(fnr)(Fnr) -> Unit—전후진 상태 갱신
onRangeShift(r)(RangeShift) -> Unit—변속 레인지 갱신
onPto(pto)(Pto) -> Unit—PTO 상태 갱신
onHydraulic(h)(Hydraulic) -> Unit—유압 피드백 갱신
onAccelerator(a)(Accelerator) -> Unit—가속 피드백 갱신
onFnrDiag(d)(FnrDiag) -> Unit—FNR 진단 갱신
onRangeShiftDiag(d)(RangeShiftDiag) -> Unit—SFT 진단 갱신
onHydraulicDiag(d)(HydraulicDiag) -> Unit—유압 진단 갱신
onAcceleratorDiag(d)(AcceleratorDiag) -> Unit—가속 진단 갱신
onStale(key)(String) -> Unit—신호 끊김(quality=DISCONNECTED) 알림 — key는 데몬 신호명

       .onRangeShift { r -> runOnUiThread { render(r.gear) } }

lifecycle.addObserver(tractor)     // 자격 불필요
