AgVehicle SDK

농기계 HMI 플랫폼의 차량 신호 SDK. 앱에서 CAN 신호를 읽고(구독) 제어(쓰기)하는 공개 창구다. 내부적으로 시스템 서비스에 bindService로 붙고, 서비스가 데몬을 거쳐 CAN 버스와 통신한다. 앱은 그 계층을 몰라도 된다.

앱 (이 SDK)──bindService──▶ agvehicled──소켓──▶ agcand──▶ CAN 버스
GETTING STARTED

Quickstart

엔진 RPM을 화면에 띄우는 최소 예시다. 의존성 한 줄 → 리스너 붙이기 → 생명주기 한 줄이면 끝이다.

의존성 추가

필요한 도메인 모듈만 넣으면 :core는 자동으로 딸려온다.

// app/build.gradle.kts
implementation("farm.agmo.vehicle:engine:0.1.0")

신호 읽기 — onXxx { } 람다 + 생명주기 한 줄

필요한 신호만 람다로 등록한다. object나 override가 필요 없다. addObserver를 호출하면 ON_START에 자동으로 구독되고 ON_STOP에 자동으로 해제된다. 콜백은 binder 스레드에서 호출되므로 UI는 runOnUiThread로 감싼다.

val engine = Engine(this, intervalMs = 200)             // 5Hz — core.html#throttle
engine.onEec1 { e -> runOnUiThread {
    rpmView.text = "${e.rpm.toInt()} rpm"
} }
lifecycle.addObserver(engine)                       // ← 이 한 줄이 구독/해제 전부

여러 신호를 한 곳에서 다루려면 리스너 방식도 동등하다(둘은 같은 일을 한다):

val engine = Engine(this, object : Engine.Listener {
    override fun onEec1(e: Eec1) = runOnUiThread { render(e.rpm) }
    override fun onTemperature(t: EngineTemperature) = runOnUiThread { render(t.coolantC) }
}, intervalMs = 200)

(선택) 제어 — 히치 올리기

쓰기(제어)는 세션을 얻어서 한다. null이면 자격이 없거나 이미 보유 중이거나 연결되지 않은 상태다.

val ctrl = Hitch.control(this) ?: return     // 매니페스트 USES_CONTROL 필요
ctrl.onLost { runOnUiThread { lockUi() } }         // 상위 계층이 뺏으면
ctrl.setPosition(100.0)                            // % — raw 변환은 SDK가 처리
ctrl.release()                                     // 정상 반납
다음 핵심 API는 core를 참고한다. 도메인별 신호는 좌측 메뉴(engine, vehicle, oem-tractor 등)에서 확인한다. 여러 앱이 같은 제어를 노리면 실행레벨(Tier)로 조정된다.
개념

개요

SDK는 기반 core 하나 + 신호 도메인별 모듈 여러 개로 나뉜다. 필요한 도메인만 가져다 쓴다. 콜백이 기본이라 코루틴 의존이 없고(Java 앱도 OK), 생명주기는 lifecycle.addObserver(...) 한 줄로 자동 연동된다.

설계 원칙: CAN 메시지(ID) 하나 = 클래스 하나. 한 PGN은 한 프레임으로 통째로 오므로, 그 안의 신호들(pitch+roll 등)은 한 데이터 클래스의 필드가 된다. 도메인 모듈은 core 위에 타입 있는 파사드를 얹은 것이다. 신호 조립, raw 변환 은닉, 자동완성 발견성 등을 제공한다.

한눈에 읽기에는 자격이 필요 없다. 쓰기(제어)는 매니페스트 자격 선언 + 제어 세션이 필요하고, 여러 앱이 같은 신호를 노릴 땐 실행레벨(Tier)로 중재된다.
GETTING STARTED

설치 (Gradle)

GitHub Packages(Maven)에서 받는다. 필요한 도메인 모듈만 넣으면 :core는 자동으로 딸려온다.

// settings.gradle.kts
dependencyResolutionManagement {
  repositories {
    maven {
      url = uri("https://maven.pkg.github.com/hobeen-agmo/agvehicle-sdk")
      credentials {           // read:packages 토큰
        username = providers.gradleProperty("gpr.user").get()
        password = providers.gradleProperty("gpr.token").get()
      }
    }
  }
}

// app/build.gradle.kts
dependencies {
  implementation("farm.agmo.vehicle:engine:0.1.0")   // :core 자동 포함
  implementation("farm.agmo.vehicle:hitch:0.1.0")
}
인증 GitHub Packages는 패키지가 public이어도 익명 다운로드를 허용하지 않는다. 소비자도 read:packages 토큰이 필요하다(GitHub 제약). 토큰 없는 공개 배포가 필요하면 JitPack / Maven Central을 검토한다.
모듈

모듈 구성

표준(공개 규격 J1939/ISO 11783을 재작성할 수 있는 모듈)과 AGMO 제조사 고유(oem)로 나뉜다. 각 모듈 카드를 눌러 상세 신호와 클래스 문서로 이동한다.

:core표준·기반
연결, 제네릭 구독/제어, 카탈로그, Signal 베이스 등을 제공한다.
AgVehicle · Subscription · ControlSession
 :engine표준
엔진 신호를 읽는다. RPM, 부하, 온도, 압력, 연료, 배기 등이 포함된다.
Eec1 · Et1 · Eflp1 · Ic1 · Lfe …
 :vehicle표준
차량 상태를 읽는다. 차속, PTO, 배터리, DPF, GPS, 변속 등이 포함된다.
Ccvs1 · Etc1/2 · Vdc2 · Vep1 …
 :hitch표준
히치 위치를 읽고 제어한다. setPosition(%)으로 raw 값을 은닉한다.
HitchPosition · HitchControl
 :imu · :gpio표준
자세(각도, 각속도, 가속도)를 읽고 범용 GPIO IO를 제공한다.
ImuAngles · ImuRates · GpioSignals
 :flow표준·옵션
위 신호들을 Kotlin Flow로 제공한다. 코루틴을 사용하는 유일한 모듈이다.
ImuFlow · EngineFlow …
 :oem-tractor🏭 AGMO
FNR, 변속, PTO, 유압, 가속 신호를 읽고 AutoLift와 히치를 제어한다.
Fnr · RangeShift · Pto · AutoLift
 :oem-steer🏭 AGMO
조향모터(Keya) 상태를 읽고 조향 명령을 내린다.
SteerAngle · SteerMotorStatus
 :oem-spreader🏭 AGMO
살포기 GNSS를 읽고 살포 모터를 제어한다.
GnssPosition · GnssTimeStatus
 :oem-imu🏭 AGMO
보조 자이로(Allynav R70)를 읽고 MTLT305를 설정한다.
GyroAngle · ImuSensorStatus
 :oem-genericio🏭 AGMO
범용 CAN IO 보드의 입력을 읽고 PWM/LED를 출력한다.
GenericIoSignals
