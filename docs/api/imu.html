IMU · GPIO (:imu · :gpio)

이 모듈은 차체 자세(각도, 각속도, 가속도)와 범용 하드웨어 GPIO 입출력을 다룬다. 둘 다 읽기에는 자격이 필요하지 않으며, 표준 계열(비-oem)에 속한다. :gpio는 CAN이 아니라 데몬의 sysfs/IIO 백엔드를 쓰는 플랫폼 하드웨어 능력이라 :imu와 별도 모듈로 나뉜다.

읽기 · 자세

:imu

"CAN 메시지(ID) 하나 = 클래스 하나" 원칙대로, IMU는 각도, 각속도, 가속도의 3개 메시지(PGN)로 온다. 필요한 콜백만 override하면 되고, 스트림별 구독과 조합이 필요하면 :flow 모듈을 쓴다.

Imu(context: Context, intervalMs: Long = 0) read farm.agmo.vehicle.imu

읽기 전용이므로 별도 자격 선언이 필요하지 않다. 콜백은 binder 스레드에서 호출되므로 UI 갱신은 앱이 runOnUiThread로 감싼다.

등록 방식은 두 가지이며, 두 방식은 기능적으로 동등하다.

방식생성 · 등록
람다Imu(context, intervalMs) + imu.onAngles { } — 필요한 신호만, 체이닝 가능
리스너Imu(context, listener: Listener, intervalMs) — 여러 신호를 한 객체에서 override

각 콜백마다 대응하는 람다 메서드가 있다. onAngles { }, onRates { }, onAccel { }가 있으며, 아래 Listener 표의 on… 이름과 1:1로 대응한다. 람다 방식은 등록한 신호만 구독한다.

intervalMs는 앱 콜백의 최소 간격(ms)이다. 0이면 신호가 올 때마다 전부 전달하고, >0이면 최대 그 간격마다 최신값을 1회만 전달한다.

Imu.Listener

콜백인자설명
onAnglesImuAngles피치·롤 각도 갱신
onRatesImuRates축별 각속도 갱신
onAccelImuAccel축별 가속도 갱신
 
val imu = Imu(this, intervalMs = 200)
imu.onAngles { a -> runOnUiThread { render(a.pitchDeg, a.rollDeg) } }
   .onRates  { r -> runOnUiThread { render(r.xDegS, r.yDegS, r.zDegS) } }
lifecycle.addObserver(imu)

여러 신호를 한 객체에서 다루려면 아래와 같이 리스너 방식으로 등록할 수도 있다. 두 방식은 기능적으로 동등하다.

val imu = Imu(this, object : Imu.Listener {
  override fun onAngles(a: ImuAngles) = runOnUiThread { render(a.pitchDeg, a.rollDeg) }
  override fun onRates(r: ImuRates)   = runOnUiThread { render(r.xDegS, r.yDegS, r.zDegS) }
}, intervalMs = 200)
lifecycle.addObserver(imu)     // ON_START 자동 구독 / ON_STOP 자동 해제

읽기 · 쓰기 · 범용 하드웨어

:gpio

:gpio는 범용 GPIO 입출력을 다룬다. 출처는 SeamOS GPIO_Prototyping 플러그인 인터페이스 이관이며, 특정 제조사 CAN 고유가 아니라 범용 하드웨어 추상화라 표준 계열(비-oem)에 둔다. 데몬 쪽 백엔드는 CAN이 아니라 sysfs/IIO다. 입력은 Gpio(Signal 상속)로 구독하고, 디지털 출력은 Gpio.digitalOut() 세션으로 제어한다.

object GpioSignals farm.agmo.vehicle.gpio

GpioSignals는 채널별 신호 키(데몬 카탈로그 키 = 신호명 그대로)와 채널 수 상수를 정의한다. 실제 핀맵(어느 출력이 무슨 릴레이인지)은 데몬 gpio_config에서 결정된다.

필드타입단위설명
ANALOG_IN_COUNTInt채널4 — 아날로그 입력(ADC) 채널 수
DIGITAL_IN_COUNTInt채널3 — 디지털 입력(DIO) 채널 수
DIGITAL_OUT_COUNTInt채널7 — 디지털 출력(DIO) 채널 수
analogIn(channel: Int)String–"Analog_IN{channel}" — 아날로그 입력 키 (1~4)
digitalIn(channel: Int)String–"Digital_IN{channel}" — 디지털 입력 키 (1~3)
digitalOut(channel: Int)String–"Digital_OUT{channel}" — 디지털 출력 키 (1~7)

Gpio — 입력 읽기

Gpio(context: Context, intervalMs: Long = 0) read farm.agmo.vehicle.gpio

읽기 전용이므로 별도 자격 선언이 필요하지 않다. channel은 1부터 시작하는 1-기반 인덱스다.

등록 방식은 두 가지이며, 두 방식은 기능적으로 동등하다.

방식생성 · 등록
람다Gpio(context, intervalMs) + gpio.onAnalogIn { } — 필요한 신호만, 체이닝 가능
리스너Gpio(context, listener: Listener, intervalMs) — 여러 신호를 한 객체에서 override

각 콜백마다 대응하는 람다 메서드가 있다. onAnalogIn { }, onDigitalIn { }가 있으며, 아래 Listener 표의 on… 이름과 1:1로 대응한다. 채널 기반이라 콜백 인자가 (channel, value)/(channel, on) 두 개다. 람다 방식은 등록한 신호만 구독한다.

intervalMs는 앱 콜백의 최소 간격(ms)이다. 0이면 신호가 올 때마다 전부 전달하고, >0이면 최대 그 간격마다 최신값을 1회만 전달한다.

콜백인자설명
onAnalogIn(channel: Int, value: Double)아날로그 입력 값 갱신
onDigitalIn(channel: Int, on: Boolean)디지털 입력 상태 갱신
 
val gpio = Gpio(this, intervalMs = 200)
gpio.onAnalogIn  { ch, v -> runOnUiThread { render(ch, v) } }
    .onDigitalIn { ch, on -> runOnUiThread { render(ch, on) } }
lifecycle.addObserver(gpio)

여러 신호를 한 객체에서 다루려면 아래와 같이 리스너 방식으로 등록할 수도 있다. 두 방식은 기능적으로 동등하다.

val gpio = Gpio(this, object : Gpio.Listener {
  override fun onAnalogIn(ch: Int, value: Double) = runOnUiThread { render(ch, value) }
  override fun onDigitalIn(ch: Int, on: Boolean)   = runOnUiThread { render(ch, on) }
}, intervalMs = 200)
lifecycle.addObserver(gpio)

Gpio.digitalOut — 디지털 출력 제어

필수 디지털 출력 제어는 매니페스트에 USES_CONTROL 선언이 필요하다(채널별 출력 key). 자세한 자격 및 실행레벨 규칙은 정책 문서를 참고한다.
Gpio.digitalOut(context: Context, channel: Int): GpioOutput? ctrl farm.agmo.vehicle.gpio

채널 1~7의 디지털 출력 제어권을 반환한다. null은 자격 없음, 세션 보유 중, 미연결, 잘못된 channel 중 하나를 의미한다.

class GpioOutput ctrl farm.agmo.vehicle.gpio
메서드타입단위설명
set(on: Boolean)Boolean–켜기(true)/끄기(false). false 반환 = 세션 상실 — digitalOut부터 다시
onLost(cb: () -> Unit)Unit–상위 계층 선점 통지 등록
release()Unit–정상 반납
stopAndRelease()Unit–안전값(off)으로 되돌린 뒤 반납
 
val out = Gpio.digitalOut(this, channel = 3) ?: return
out.set(true)      // Digital_OUT3 ON. false=세션 상실
out.release()
