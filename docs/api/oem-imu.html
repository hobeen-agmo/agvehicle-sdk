:oem-imu

🏭 AGMO 제조사 고유 모듈이다. 보조 자이로(Allynav R70)의 자세각, 가속도, 각속도를 읽고 Aceinna MTLT305의 진단(센서 health, 펌웨어 버전 등)을 제공한다. 표준 자세 데이터(각도/각속도/가속도)는 표준 :imu 모듈이 담당하며, 이 모듈은 그것으로 커버되지 않는 제조사 고유 신호만 다룬다.

사용법

빠른 시작

필요한 신호만 onXxx { } 람다로 등록한다. object 선언이나 override 없이 등록할 수 있으며 체이닝도 가능하다. intervalMs로 앱 콜백의 최소 간격(ms)을 조절한다.

val gyro = OemImu(this, intervalMs = 200)
gyro.onAngle { a -> runOnUiThread { render(a.xDeg, a.yDeg, a.zDeg) } }
    .onRate  { r -> runOnUiThread { render(r.zDegS) } }
lifecycle.addObserver(gyro)

여러 신호를 한 객체에서 다루려면 리스너 방식도 동등하게 쓸 수 있다.

val gyro = OemImu(this, object : OemImu.Listener {
    override fun onAngle(a: GyroAngle) = runOnUiThread { render(a.xDeg, a.yDeg, a.zDeg) }
    override fun onRate(r: GyroRate)   = runOnUiThread { render(r.zDegS) }
}, intervalMs = 200)
lifecycle.addObserver(gyro)
표준 자세 차량 표준 자세각/각속도/가속도가 필요하면 :imu를 쓴다. 이 모듈은 보조 자이로(Allynav R70)와 MTLT305 진단만 노출하며 표준과 섞지 않는다.

데이터 클래스

읽기 — MTLT305 설정/진단

Aceinna MTLT305 센서의 health, 펌웨어 정보 등을 제공한다. 표준 자세 데이터는 표준 :imu가 담당한다.

ImuSensorStatus read aceinna_mtlt305 0x18FF5480

MTLT305 센서 health를 나타낸다. 각 비트의 정상/결함 극성은 Aceinna MTLT305 데이터시트를 기준으로 한다.

필드타입단위설명
masterBoolean—마스터 상태
hardwareBoolean—하드웨어 상태
softwareBoolean—소프트웨어 상태
sensorBoolean—센서 상태
algoInitBoolean—알고리즘 초기화 상태
overRangeBoolean—센서 측정 범위 초과

KEYS 접두어: aceinna_mtlt305:Aceinna_<필드>_Status 계열 (예: aceinna_mtlt305:Aceinna_Master_Status, aceinna_mtlt305:Aceinna_Sensor_Over_Range)

ImuFirmwareVersion read aceinna_mtlt305 0x18FEDA80

MTLT305 펌웨어 버전이다. version은 "major.minor.patch" 형태의 계산 프로퍼티다.

필드타입단위설명
majorInt—메이저 버전
minorInt—마이너 버전
patchInt—패치 버전
stageInt—빌드 스테이지
buildInt—빌드 번호
versionString (get)—"$major.$minor.$patch"

KEYS: aceinna_mtlt305:Aceinna_Firm_Version_Major, _Minor, _Patch, _Stage, _Build

보류 MTLT305 설정(쓰기) API는 아직 없다. 현재 이 모듈은 진단(health, 펌웨어 버전 등) 읽기만 제공하며, 설정 API는 후속으로 추가될 예정이다.
API

OemImu.Listener

OemImu(context: Context, intervalMs: Long = 0) read :oem-imu

등록 방식은 두 가지이며, 두 방식은 동등하다.

방식생성 · 등록
람다OemImu(context, intervalMs) + gyro.onAngle { } — 필요한 신호만, 체이닝 가능
리스너OemImu(context, listener: Listener, intervalMs) — 여러 신호를 한 객체에서 override

각 콜백마다 대응하는 람다 메서드가 있다. onAngle { }, onAccel { }, onRate { }, onSensorStatus { }, onFirmwareVersion { } 등이 있으며, 아래 Listener의 on… 이름과 1:1로 대응한다. 람다 방식은 등록한 신호만 구독한다.

intervalMs는 앱 콜백의 최소 간격(ms)을 지정한다. 0이면 신호가 올 때마다 전부 전달하고, >0이면 최대 그 간격마다 최신값을 1회만 전달한다.

interface Listener {
    // Allynav R70 보조 자이로
    fun onAngle(angle: GyroAngle) {}
    fun onAccel(accel: GyroAccel) {}
    fun onRate(rate: GyroRate) {}
    // Aceinna MTLT305 설정/진단 (자세 데이터는 표준 :imu)
    fun onSensorStatus(status: ImuSensorStatus) {}
    fun onFirmwareVersion(fw: ImuFirmwareVersion) {}
}
