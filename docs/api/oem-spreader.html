:oem-spreader

🏭 AGMO 제조사 고유 모듈이다. 살포기(RDA Spreader)의 GNSS 위치, 시간, 상태를 읽고 살포 모터 구동을 제어한다. 살포 모터는 DriveMode와 MotorEnable을 한 프레임(MotorCommand)에 결합해 원자적으로 지령한다.

사용법

빠른 시작

GNSS 읽기는 자격이 필요 없다. 살포 모터 제어는 매니페스트에 USES_CONTROL 선언이 필요하다.

val spreader = Spreader(this, intervalMs = 200)
spreader.onPosition   { p -> runOnUiThread { render(p.latitudeDeg, p.longitudeDeg) } }
        .onTimeStatus { t -> runOnUiThread { render(t.fix) } }
lifecycle.addObserver(spreader)

여러 신호를 한 객체에서 다루려면 리스너 방식도 동등하다.

// GNSS 읽기 (자격 불필요)
val spreader = Spreader(this, object : Spreader.Listener {
    override fun onPosition(p: GnssPosition) = runOnUiThread { render(p.latitudeDeg, p.longitudeDeg) }
    override fun onTimeStatus(t: GnssTimeStatus) = runOnUiThread { render(t.fix) }
}, intervalMs = 200)
lifecycle.addObserver(spreader)

// 살포 모터 제어 (매니페스트에 USES_CONTROL 선언 필요)
val motor = Spreader.motorControl(this) ?: return
motor.set(DriveMode.SPREADING, enable = true)   // 모드+enable 원자적 송신
motor.set(DriveMode.STOP, enable = false)
motor.release()
클래스

Spreader

생성자로 붙이는 즉시 GNSS 구독 대상이 정해진다. 실제 구독/해제는 lifecycle.addObserver(spreader)가 ON_START/ON_STOP에 맞춰 처리한다.

Spreader(context: Context, intervalMs: Long = 0) read agmo_spreader

등록 방식은 두 가지이며, 둘은 동등하다.

방식생성 · 등록
람다Spreader(context, intervalMs) + spreader.onPosition { } — 필요한 신호만, 체이닝 가능
리스너Spreader(context, listener: Listener, intervalMs) — 여러 신호를 한 객체에서 override

각 콜백마다 대응하는 람다 메서드가 있다. onPosition { }, onTimeStatus { } 등이 있으며, 아래 Listener 표의 on… 이름과 1:1로 대응한다. 람다 방식은 등록한 신호만 구독한다.

intervalMs는 앱 콜백의 최소 간격(ms)을 지정한다. 0이면 신호가 올 때마다 전부 전달하고, >0이면 최대 그 간격마다 최신값을 1회만 전달한다.

GNSS 위경도를 나타낸다. raw × 1e-7(DBC 정의 그대로) 값이며, 데몬이 스케일을 적용해 전달한다.

제어

쓰기 — 살포 모터 제어

Spreader.motorControl(context)로 제어 세션을 얻는다. null이면 자격 없음, 보유 중, 미연결 중 하나다. 매니페스트에 USES_CONTROL 선언이 필요하다.

SpreaderMotorControl ctrl agmo_spreader MotorCommand (16bit)

모드(DriveMode)와 enable을 한 프레임에 결합해 원자적으로 송신한다. 계산식은 raw = mode.code | (enable ? 1<<8 : 0)이다.

메서드반환설명
set(mode: DriveMode, enable: Boolean)Boolean모드+enable 지령. false=세션 상실 — motorControl부터 다시
onLost(cb: () -> Unit)Unit상위 계층에 제어권을 뺏겼을 때 콜백
release()Unit제어권 정상 반납
stopAndRelease()Unit정지 지령 후 반납
