Engine

:engine은 엔진 신호를 읽는 도메인 모듈이다. RPM, 부하, 토크, 온도, 압력, 연료, 흡배기, DPF, DEF 등 여러 J1939 메시지에 흩어져 있는 엔진 신호를 메시지(ID) 하나에 클래스 하나 원칙으로 묶는다. 이렇게 정의한 데이터 클래스 17종과, 이들을 구독하는 Engine 및 Engine.Listener를 함께 제공한다. 모든 신호가 읽기 전용이므로 별도의 자격 선언은 필요하지 않다.

주의 Eec1.loadPercent는 이름과 달리 EEC1(0xF004)이 아니라 EEC2(0xF003, SPN 92)에 속한다. 데몬 agcand의 signal_defs.cpp를 기준으로 ENGRPM만 0xF004이고 ENGLOAD는 0xF003이다. 이는 "CAN 메시지 하나에 클래스 하나" 원칙에 어긋나지만, 공개 API 하위호환을 위해 필드를 그대로 남겨 두었으며 @Deprecated로 표시했다. 신규 코드에서는 Eec2.loadPercent를 사용한다.
사용 예제

Engine + lifecycle

필요한 신호만 onXxx { } 람다로 등록한다. object 선언이나 메서드 override 없이 등록할 수 있고, 여러 메서드를 이어서 호출(체이닝)할 수 있다. intervalMs 인자로 앱 콜백의 최소 간격(ms)을 지정한다.

val engine = Engine(this, intervalMs = 200)               // 5Hz — core.html#throttle
engine.onEec1 { e -> runOnUiThread { rpmView.text = "${e.rpm.toInt()} rpm" } }
      .onEec2 { e -> runOnUiThread { loadView.text = "${e.loadPercent.toInt()} %" } }
      .onTemperature { t -> runOnUiThread { coolantView.text = "${t.coolantC.toInt()} °C" } }
lifecycle.addObserver(engine)                         // ON_START 구독 / ON_STOP 해제

여러 신호를 한 객체에서 처리하려면 아래와 같이 리스너 방식으로 등록할 수도 있다. 두 방식은 기능적으로 동등하다.

val engine = Engine(this, object : Engine.Listener {
    override fun onEec1(e: Eec1) = runOnUiThread { render(e.rpm) }
    override fun onTemperature(t: EngineTemperature) = runOnUiThread { render(t.coolantC) }
}, intervalMs = 200)
클래스

Engine

생성자에 콜백을 등록하는 즉시 구독 대상이 정해진다. 실제 구독과 해제는 lifecycle.addObserver(engine)가 ON_START와 ON_STOP 시점에 맞춰 처리한다.

Engine(context: Context, intervalMs: Long = 0) read :engine

등록 방식은 두 가지이며, 두 방식은 기능적으로 동등하다.

방식생성 · 등록
람다Engine(context, intervalMs) + engine.onEec1 { } — 필요한 신호만, 체이닝 가능
리스너Engine(context, listener: Listener, intervalMs) — 여러 신호를 한 객체에서 override

콜백마다 대응하는 람다 메서드가 있다. onEec1 { }, onEec2 { }, onTemperature { }, onOilPressure { } 등이 있으며, 아래 Listener 표의 on… 이름과 1:1로 대응한다. 람다 방식은 등록한 신호만 구독한다.

intervalMs는 앱 콜백의 최소 간격(ms)을 지정한다. 값이 0이면 신호가 올 때마다 모두 전달하고, 0보다 크면 그 간격마다 최신값을 한 번만 전달한다.

콜백은 binder 스레드에서 호출되므로 UI 갱신은 runOnUiThread로 감싼다.

메시지별 데이터 클래스

각 클래스는 하나의 J1939 메시지(PGN)에 대응한다. KEYS는 core.html의 신호 키 목록이며, from(map) 팩토리로 Map<String, Double>에서 조립한다(필요한 키가 하나라도 없으면 null).

Eec1 read 0xF004 · SPN 190 :engine

엔진 회전수(EEC1). onEec1 콜백, KEYS = ["ENGRPM", "ENGLOAD"], Eec1.from(map).
