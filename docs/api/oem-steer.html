:oem-steer

🏭 AGMO 제조사 고유 조향모터(Keya KY170) 상태 읽기 도메인이다. Motor_Heartbeat로 결함, 상태를 받고 Motor_Response의 CANopen SDO(Data_ID 멀티플렉서)로 조향각, 속도, 온도, 전압 등을 받는다. 조향 명령(쓰기)은 아직 없다.

사용법

빠른 시작

Steer는 Signal을 상속한 읽기 전용 도메인이다. 필요한 신호만 onXxx { } 람다로 등록한다. object 선언이나 override 없이 등록할 수 있고, 체이닝도 가능하다. intervalMs로 앱 콜백 최소 간격(ms)을 조절한다.

val steer = Steer(this, intervalMs = 200)
steer.onMotorStatus { s -> runOnUiThread { if (s.fault) warn(s) } }
     .onAngle       { a -> runOnUiThread { render(a) } }
lifecycle.addObserver(steer)

여러 신호를 한 객체에서 다루려면 아래와 같이 리스너 방식으로 등록할 수도 있다. 두 방식은 기능적으로 동등하다.

val steer = Steer(this, object : Steer.Listener {
    override fun onMotorStatus(s: SteerMotorStatus) = runOnUiThread { if (s.fault) warn(s) }
}, intervalMs = 200)
보류 조향 명령(Motor_Request 0x06000001)은 CANopen SDO 멀티플렉서 코드별 payload의 정식 레이아웃이 확보되지 않아 아직 미노출이다. 현재 :oem-steer는 읽기 API만 제공한다.
데이터 클래스

읽기 — 조향모터 상태

SteerMotorStatus read keya_ky170 0x07000001 (Motor_Heartbeat)

조향모터 상태와 결함을 나타낸다. 각 플래그는 해당 결함이 발생했을 때 true다. fault는 플래그 중 하나라도 발생하면 true인 계산 프로퍼티다.

필드타입단위설명
hallFailureBoolean—홀센서 결함
canDisconnectedBoolean—CAN 연결 끊김
motorStalledBoolean—모터 정지(스톨)
disabledBoolean—모터 비활성화 상태
overvoltageBoolean—과전압
hardwareProtectionBoolean—하드웨어 보호 동작
e2promBoolean—E2PROM 결함
undervoltageBoolean—저전압
overcurrentBoolean—과전류
modeFailureBoolean—모드 결함
faultBoolean (get)—위 플래그 중 하나라도 true면 true

KEYS 접두어: keya_ky170:Heartbeat_<필드명> (예: keya_ky170:Heartbeat_HallFailure)

데이터 클래스

읽기 — 조향각·속도·온도·전압

아래 네 값은 모두 Motor_Response(0x05800001) CANopen SDO에서 Data_ID 멀티플렉서로 개별 프레임에 나뉘어 도착한다. 데몬이 mux를 지원해 각각 독립 신호로 분리해 준다.

SteerAngle read keya_ky170 0x05800001 (SDO · EncoderCountValue)

API

Steer.Listener

Steer(context: Context, intervalMs: Long = 0) read :oem-steer

등록 방식은 두 가지이며, 둘은 동등하다.

방식생성 · 등록
람다Steer(context, intervalMs) + steer.onMotorStatus { } — 필요한 신호만, 체이닝 가능
리스너Steer(context, listener: Listener, intervalMs) — 여러 신호를 한 객체에서 override

각 콜백마다 대응하는 람다 메서드가 있다. onMotorStatus { }, onAngle { }, onSpeed { }, onTemperature { }, onVoltage { } 등이 있으며, 아래 Listener의 on… 이름과 1:1로 대응한다. 람다 방식은 등록한 신호만 구독한다.

intervalMs는 앱 콜백의 최소 간격(ms)을 지정한다. 0이면 신호가 올 때마다 전부 전달하고, 0보다 크면 최대 그 간격마다 최신값을 1회만 전달한다.

필요한 콜백만 override한다. 모든 콜백은 binder 스레드에서 호출되므로 UI 갱신은 runOnUiThread로 감싼다.

interface Listener {
    fun onMotorStatus(status: SteerMotorStatus) {}
    fun onAngle(angle: SteerAngle) {}
    fun onSpeed(speed: SteerSpeed) {}
    fun onTemperature(temperature: SteerTemperature) {}
    fun onVoltage(voltage: SteerVoltage) {}
}
